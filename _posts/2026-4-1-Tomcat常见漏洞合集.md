# Tomcat常见漏洞合集



Apache Tomcat 是一个 **Java Web 服务器** 类似于 Python 中的 Flask 或 PHP 的 Apache+PHP

一般目录结构：

```
tomcat/
├── bin/
├── conf/
├── logs/
├── webapps/
│   ├── ROOT/
│   ├── manager/
│   └── examples/
```


## PUT方法任意文件写入漏洞  CVE-2017-12615（tomcat 7.0.0 – 7.0.81 部分8.5.19)

当 Tomcat 开启了 PUT 写文件功能且配置不当时，攻击者可通过 HTTP PUT 上传 JSP，Tomcat 再将其当动态脚本执行，最终 RCE。



**关键触发条件：**

readonly 参数设置为 false；
目标环境支持 HTTP PUT 方法；
攻击者能绕过文件名后缀限制（如 .jsp）。



**漏洞原理：**

1. Tomcat 支持 HTTP PUT

HTTP PUT 方法可用于向服务器指定路径写入资源。

------

2. 配置不当允许 PUT 写入 Web 目录

若 PUT 写入位置位于可被 Web 访问并解析的目录（如 webapps），上传内容可能成为可访问文件。

------

3. 路径校验缺陷

受影响版本中，Tomcat 对如下路径处理存在缺陷：

```
shell.jsp/
```

校验阶段将其视为**目录路径**，未正确拦截 `.jsp` 上传；
 但落盘时实际写入为：

```
shell.jsp
```

从而绕过 JSP 文件上传限制。


## AJP协议文件包含漏洞（CVE-2020-1938，Ghostcat）

Tomcat 的 AJP 协议配置不当时，攻击者可通过 AJP 协议读取 Web 应用文件，特定场景下甚至进一步 RCE。



**影响版本**
以下版本的Apache Tomcat受此漏洞影响：

Tomcat 6.x：全版本
Tomcat 7.x：7.0.0 ≤ 版本 < 7.0.100
Tomcat 8.x：8.0.0 ≤ 版本 < 8.5.51
Tomcat 9.x：9.0.0 ≤ 版本 < 9.0.31



AJP 全称

> **Apache JServ Protocol**(默认端口8009)

它是：

> **Web Server 和 Tomcat 之间通信的内部协议**

前端用 Apache HTTP Server / Nginx 做反向代理

后端用 Apache Tomcat 跑 Java 应用

二者通过 AJP 通信



**关键触发条件**：

1. 目标Tomcat服务开启AJP协议端口（默认8009）；
2. 未配置AJP协议认证（如`secret`或`requiredSecret`参数）。



**复现流程：**

`docker compose up -d`

开启容器



用脚本包含任意文件：

`python poc.py 127.0.0.1 -p 8009 -f WEB-INF/web.xml`

## Tomcat弱口令与后台WAR包部署漏洞



**利用条件**

1. 配置缺陷

   ：

   - 管理后台允许远程访问（默认仅限本地访问，需手动修改 `context.xml` 解除限制）。
   - 存在弱口令或默认凭据（如 `tomcat/tomcat`、`admin/admin` 或空密码）。

2. 功能启用

   ：

   - 用户角色需具备 `manager-gui` 或 `manager-script` 权限以访问部署功能。



**复现过程：**



打开tomcat管理页面`http://your-ip:8080/manager/html`，输入弱密码`tomcat:tomcat`，即可访问后台：

![](https://cdn.jsdelivr.net/gh/gola-leya/img-bed/img/20260411222522.png)



后台可以上传war文件(war文件可以理解成 Java Web 的压缩部署包)



如何准备一个一个war包？

1. 先准备了一个JSP的一句话木马，安装好JDK环境
2. 执行:`jar -cvf [war包名称].war 打包目录`
